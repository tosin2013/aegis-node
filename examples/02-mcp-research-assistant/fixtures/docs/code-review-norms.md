# Code Review Norms

Reviewers look for: correctness, clarity, and consistency with the rest
of the codebase, in that order. Style nits are fine but never block a PR.

We aim for review turnaround under 4 working hours during the team's
overlap window. If a PR sits longer than 24h waiting for review, the
author can ping any reviewer or merge with the team lead's sign-off.

Disagreements escalate to the team lead, not to lengthy thread debates.
